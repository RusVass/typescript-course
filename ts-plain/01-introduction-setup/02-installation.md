# Встановлення TypeScript у проєкт

## Глобальне встановлення TypeScript

Глобальне встановлення дозволяє використовувати TypeScript у будь-якому проєкті на вашому комп’ютері. Це зручно, якщо ви часто працюєте з TypeScript і хочете мати доступ до його інструментів з командного рядка будь-де.

### Встановлення глобально

```bash
npm install -g typescript
```

Після встановлення ви зможете використовувати команду `tsc` для компіляції файлів TypeScript з будь-якого місця.

**Переваги:**
- Зручно використовувати в різних проєктах.
- Не потрібно встановлювати TypeScript у кожному проєкті окремо.

**Недоліки:**
- Можливі конфлікти версій, якщо різні проєкти вимагають різні версії TypeScript.

## Локальне встановлення TypeScript

Локальне встановлення TypeScript у проєкті дає змогу керувати версією TypeScript, що використовується в конкретному випадку. Це особливо корисно, якщо ваш проєкт потребує певної версії або якщо ви працюєте в команді, де важливо, щоб усі використовували однакову версію.

### Встановлення локально

```bash
npm install --save-dev typescript
```

Після встановлення ви можете використовувати команду `npx tsc` для компіляції файлів TypeScript у контексті свого проєкту.

**Переваги:**
- Керування версіями TypeScript для кожного проєкту.
- Уникнення конфліктів версій між проєктами.
- Легше підтримувати узгодженість у команді.

**Недоліки:**
- Потрібно встановлювати TypeScript у кожному проєкті окремо.

## Рекомендації

Вибір між глобальним і локальним встановленням залежить від ваших потреб та вподобань. У більшості випадків варто використовувати локальне встановлення, щоб уникнути проблем із сумісністю версій і забезпечити узгодженість у команді.


## Що таке tsconfig.json і як його згенерувати

`tsconfig.json` — це конфігураційний файл для компілятора TypeScript. Він визначає кореневі файли та параметри компіляції, необхідні для проєкту. Використання `tsconfig.json` дає змогу налаштувати поведінку компілятора відповідно до ваших потреб.

### Генерація tsconfig.json

Щоб згенерувати файл `tsconfig.json`, виконайте в корені свого проєкту команду: `npx tsc --init`
